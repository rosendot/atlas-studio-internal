# Client Preflight

Reads the `preflight-report.md` generated by the Theme Preflight agent and applies fixes, replacing placeholder content with client-specific values.

## Usage
`/client-preflight <theme-slug> <client-name>`

## Prerequisites
The Theme Preflight agent must have been run first. This skill reads `preflight-report.md` from the theme directory. If the report does not exist, tell the user:

> Run the Theme Preflight agent on this theme first. It will generate the `preflight-report.md` that this skill needs.

## Steps

1. Read `themes/$ARGUMENTS/preflight-report.md` (parse theme-slug and client-name from $ARGUMENTS).
2. Check the **Launch Readiness** status.

### Phase 1: Gather client values
3. Read the **Placeholder Values to Confirm** table from the report.
4. Present the table to the user and ask them to provide actual values for each placeholder:
   ```
   The preflight report found these placeholder values that need client data:
   
   - business_name: "Restaurant Name" → ?
   - phone: "(555) 000-0000" → ?
   - email: "hello@example.com" → ?
   - address: "123 Main Street" → ?
   ...
   
   Please provide the actual values for this client.
   ```
5. Wait for the user to provide values before proceeding.

### Phase 2: Replace placeholders
6. For each placeholder value the user provided:
   - Find every occurrence in the theme files (the report lists files and lines)
   - Replace with the client-provided value
   - Ensure replacements are properly escaped in PHP context
7. Update `theme.json` variables with the new default values.

### Phase 3: Fix blockers and warnings
8. Parse all **Blocker** and **Warning** issues from the report.
9. For each issue:
   - Read the referenced file
   - Apply the fix described in the report
   - Verify the fix doesn't break surrounding code
10. Apply all Blockers and Warnings without asking.

### Phase 4: Handle notes
11. List the **Notes** and ask the user which ones they want applied.

### Phase 5: Cleanup
12. Delete `preflight-report.md` — it is now stale.
13. Summarize what was changed:
    - Placeholder values replaced (with counts)
    - Blockers fixed
    - Warnings fixed
    - Notes skipped or applied
    - Remaining items that need manual attention

## Fix Patterns

### Placeholder text replacement
```php
// Before
<h1><?php echo esc_html( 'Restaurant Name' ); ?></h1>
// After
<h1><?php echo esc_html( 'Tacoria' ); ?></h1>
```

### Theme.json variable update
```json
// Before
"business_name": {
  "label": "Business Name",
  "type": "string",
  "default": "Restaurant Name"
}
// After
"business_name": {
  "label": "Business Name",
  "type": "string",
  "default": "Tacoria"
}
```

### Missing 404 template
Create `404.php`:
```php
<?php get_header(); ?>
<main class="error-404">
	<h1><?php esc_html_e( 'Page not found', 'theme-slug' ); ?></h1>
	<p><?php esc_html_e( 'The page you are looking for does not exist.', 'theme-slug' ); ?></p>
	<a href="<?php echo esc_url( home_url( '/' ) ); ?>">
		<?php esc_html_e( 'Return home', 'theme-slug' ); ?>
	</a>
</main>
<?php get_footer(); ?>
```

### Remove debug code
Delete any `console.log()`, `var_dump()`, `print_r()`, `error_log()` calls found in theme files.

### Remove hardcoded local URLs
```php
// Before
<a href="http://clientname.local/menu">
// After
<a href="<?php echo esc_url( home_url( '/menu' ) ); ?>">
```

## Rules
- ALWAYS ask the user for client values before replacing placeholders — never guess business names, phone numbers, or addresses
- Apply Blockers and Warnings without asking. Ask before applying Notes.
- All replacements must be properly escaped in their PHP context
- Update `theme.json` variables to match the new values
- Delete `preflight-report.md` when done
- If the report says "Launch Readiness: NOT READY" and there are unresolved blockers after fixes, warn the user
- After completion, recommend running the dedicated SEO/a11y/perf analyzers for a deeper check if the preflight found issues in those areas
